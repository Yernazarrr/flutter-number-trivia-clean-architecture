## рџЏ›пёЏ Architecture Layers

### 1. Domain Layer _(innermost вЂ” pure Dart)_

The **heart** of the application. Contains:

- `Entities` вЂ” core business objects
- `Repositories` вЂ” abstract contracts
- `Use Cases` вЂ” single-responsibility business logic units

### 2. Data Layer

Implements domain contracts. Contains:

- `Models` вЂ” serializable versions of entities
- `Data Sources` вЂ” remote (API) and local (cache)
- `Repository Implementations`

### 3. Presentation Layer

UI + state management. Contains:

- `BLoC` вЂ” business logic component
- `Pages` вЂ” full screens
- `Widgets` вЂ” reusable UI components

> [!IMPORTANT]
> Dependencies always point **inward**. The domain layer has zero dependencies on Flutter or any external packages.

## рџљЂ Getting Started

### Prerequisites

- Flutter SDK `>=3.0.0`
- Dart SDK `>=3.0.0`
- Android Studio / VS Code

### Installation

**1. Clone the repository**

```bash
git clone https://github.com/Yernazarrr/flutter-number-trivia.git
cd flutter-number-trivia
```

**2. Install dependencies**

```bash
flutter pub get
```

**3. Run code generation**

```bash
dart run build_runner build --delete-conflicting-outputs
```

**4. Run the app**

```bash
flutter run
```

## рџ§  Key Concepts Demonstrated

- **Clean Architecture** вЂ” strict separation of domain, data, and presentation layers
- **TDD** вЂ” test-first development with `mockito` and `bloc_test`
- **BLoC pattern** вЂ” reactive, stream-based state management
- **Either monad** вЂ” typed error handling via `fpdart` without exceptions
- **Dependency Injection** вЂ” automatic dependency registration via `injectable` + `get_it`
- **Code Generation** вЂ” `freezed`, `retrofit`, `json_serializable` to eliminate boilerplate
